# تعليمات استرجاع النسخة الاحتياطية للواجهة (UI)

تم أخذ نسخة احتياطية من حالة المشروع الحالية (بما فيها الواجهة القديمة) عن طريق إنشاء فرع (Branch) جديد في Git باسم `ui-backup-main`.
أنت الآن تعمل على الفرع الأساسي `main`، ويمكنك البدء في تعديل الواجهة بأمان.

## طرق الاسترجاع (إذا أردت العودة للواجهة القديمة):

### 1. الاسترجاع الفوري (قبل الحفظ/Commit)
إذا قمت بتعديل الملفات ولم تعجبك النتيجة وأردت مسح التعديلات فوراً (بشرط عدم عمل Commit):
```bash
git restore .
```
*(هذا الأمر سيمسح أي تعديلات غير محفوظة ويرجع الملفات لحالتها الأصلية في آخر Commit).*

### 2. الاسترجاع الكامل (بعد الحفظ/Commit)
إذا كنت قد قمت بعمل Commit لتعديلاتك الجديدة، ولكنك قررت التخلص منها تماماً والرجوع لحالة النسخة الاحتياطية:
```bash
git reset --hard ui-backup-main
```
*(تحذير: هذا الأمر سيمسح كل التعديلات (Commits) الجديدة التي قمت بها في الفرع الأساسي ويرجعه للحالة القديمة تماماً).*

### 3. إلقاء نظرة على الواجهة القديمة (مع الاحتفاظ بالجديدة)
إذا أردت فقط تشغيل الكود القديم لرؤية الواجهة السابقة بدون أن تخسر التعديلات الجديدة التي قمت بها:
- أولاً، تأكد من عمل Commit لتعديلاتك الحالية.
- ثم انتقل لفرع النسخة الاحتياطية بهذا الأمر:
```bash
git checkout ui-backup-main
```
- للعودة لعملك الجديد مرة أخرى، اكتب `git checkout main`.
